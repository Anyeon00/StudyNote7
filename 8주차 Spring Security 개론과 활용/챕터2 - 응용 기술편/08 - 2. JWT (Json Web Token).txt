  - JWT (Json Web Token)]


JWT (Json Web Token)
- JWT는 Stateless 상태를 유지하며, 서버에서 사용자를 식별할 수 있는 수단을 제공

- 구조
Header : 토큰 타입(JWT), 사용 알고리즘
Payload : 전달 데이터 (토큰발급자, 발급일자, 만료일자, 기타등등)
Signature : 위변조여부 검증 데이터 (서버만 아는 key가 포함되어 인코딩되어 있음)
* "."으로 구분하여, 통째로 base64로 인코딩하여 헤더에 담음
* 따라서 디코딩 가능하므로, 비밀번호 같은 민감정보 포함하면 안됨

- 흐름
사용자가 인증요청 - 서버가 인증 후, JWT토큰 응답
사용자가 JWT토큰을 담아 리소스 요청 - 서버가 JWT로 사용자를 식별 후 응답

- 장점
세션 클러스터가 필요없고, 확장에 용이 *세션클러스터(7-2참고)
- 단점
토큰 만료 처리가 어려움 
(세션은 그냥 서버에서 세션만료시켜버리면 되는데, JWT 토큰은 사용자가 쥐고 있으므로)


*Session vs JWT
- Stateful vs Stateless
- 서버에 저장된 세션을 인증할 때 사용하는 무의미한 key값을 전달 vs 사용자 인증정보를 전달
